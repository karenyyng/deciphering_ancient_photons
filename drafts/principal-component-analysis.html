Principal Component Analysis

Date Mon 27 July 2015

This is the beginning of a series of me trying to explain Stat / Machine-learning techniques / concepts in a succinct fashion.

Aspect Description
What it is Dimensionality reduction technique
Layman explanation data compression by picking out the most dominant features
Type Unsupervised
Steps 1. Perform eigenvalue decomposition of the mean-subtracted covariance matrix (d x d)
2. Sort eigenvectors (d of them) according to the descending order of the corresponding eigenvalues
3. Variance explained by each component is captured by the relative eigenvalue of that component , i.e. $ abs(\lambda_i) / \sum_i abs(\lambda_i) $
4. Pick enough components to explain most of the variance and discard the rest of the components, hence reducing the dimension of the data
Pros * pick up independent linear components
* should be numerically stable for most data unless there is serious collinearity
between features such that the covariance matrix is ill-conditioned
Cons * only works for picking out linear components but not nonlinear components
* computationally intensive and scales as \(O(d^3)\)
* have to recompute all data if there is new data
* sensitive to scaling of different dimensions
