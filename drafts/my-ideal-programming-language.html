My ideal programming language

Date Wed 02 September 2015 Tags Programming

There have been quite a lot of language wars going on in Data Science about which language will reign supreme.

After using some of the most popular languages for Data Science (Python, R, Matlab etc.), I do not think any of them is superior than the other. It depends on the use case. Of course, there is always a package in a certain language that will do a certain job with one command. I hope that is not the reason why people declare their language(s) as superior than the other.

I wish the discussion would gear more towards good language designs instead of talking about petty features that one language has / does not have. To me a good programming language should:

  • encourage good programming practices, e.g.
  • have good test suites for test-driven development
  • have good continuous-integration support for larger projects
  • encourage users to write small function chunks instead of spaghetti code with complicated loops and condition statements
  • good style guide for users so they can write consistent code. Code from different users should not look like a bunch of random words typed by monkeys. Should classes be written in camelCase or PascalCase? And should variables contain "." in the names when every other major language use '.' to indicate a member variable / method? These details affect readability.
  • have packages for generating good documentation
  • have some syntax guide for what should go into the documentation I do not need to know the entire related history of the function. I do need to know the type of the variables that should be used as inputs / what is output. Also if the function computes something according to some math equation. The actual equation / paper reference shoudl be documented somewhere. There is no way that the user need to parse the equation from your code for debugging.
  • have good profiling tools

  • good community to answer coding question. Don't just tell beginners to RTFM.

  • state the programming paradigms and give examples

  • have a good debugger
  • be clear when variables are passed by references and / or values and please include this in the lesson 101 of your language

And I would go further to say that I prefer languages to have the following features for computing at scale: type safe scalable - can handle multi-threads / concurrency without much modification to code * fast - both in terms of development and run time. Life is too short to be used for debugging / waiting for compilation / code running. There is important science to be done.

Lastly, I wonder when we will have a smart handling of function variables, to have a smart persistent data structure (e.g. version controled). Please do not copy variables when they have not changed - just make copies of the parts of my data structure that have changes. Thank you. And of course this feature does not seem to exist in any existing languages.

The next programming language that I will learn is probably Scala. (Even though I am also seduced by Julia) Though learning Scala, I would like to learn more about the functional programming paradigm, and the concepts behind how concurrent / easily parallel code can be written.
